# AI/ML Documentation — PINERP

Two distinct AI systems power this platform: a rule-based **decision agent** and two **trained machine learning models**. They are architecturally different and should not be conflated — see classification table at the bottom.

---

## 1. AI Procurement Optimization Agent

**Type:** Classical/symbolic AI — a rule-based autonomous decision agent (multi-criteria decision analysis). **Not machine learning** — no training data, no learned parameters.

**What it does:** Detects understocked products, scores every candidate vendor on a weighted price/speed/reliability formula computed from live data, and recommends (and can execute) the optimal purchase.

**Pipeline:** `PERCEIVE → GATHER → SCORE → DECIDE`

| Step | Logic |
|---|---|
| PERCEIVE | Product below `lowStockThreshold`, OR flagged by an active Market Signal (shortage/availability report) |
| GATHER | Live `VendorOffer` rows (price, lead time) + real `VendorQualityIncident` counts per vendor |
| SCORE | `score = priceScore×W_price + speedScore×W_speed + reliabilityScore×W_reliability` (weights user-adjustable via UI sliders, normalized to sum to 1) |
| DECIDE | Highest-scoring vendor wins; full reasoning trace returned, not a black box |

**Files:**
| File | Role |
|---|---|
| `backend/src/engines/procurementOptimizationAgent.ts` | Core agent — scoring, decision logic, PO execution |
| `backend/src/modules/insights/insights.routes.ts` | API endpoints (`/optimize-procurement`, `/vendor-comparison/:productId`) |
| `backend/src/prisma/addOptimizationSeedData.ts` | Vendor offers + quality incident seed data the agent reads |

**Data it runs on:** Live production data — `Product`, `VendorOffer`, `VendorQualityIncident`, `MarketSignal` tables. Nothing pretrained; recomputes fresh on every call.

**UI:** Insights page → "AI Procurement Optimization Agent" card. Pick a product → adjust weight sliders → Run Optimization → Apply (creates a real Purchase Order + Audit Log entry).

## 2a. Demand Forecast — Linear Regression

**Type:** Classical machine learning (simplest form — 2 parameters fit from data via least squares).

**What it does:** Fits a trend line to a product's real monthly Stock Ledger sales history, projects 3 months forward, self-reports confidence via R².

**Files:**
| File | Role |
|---|---|
| `backend/src/engines/forecastingEngine.ts` | `fitLinearRegression()` — OLS implementation + forecast generation |

**Training data:** The product's own real sales history (from `StockLedger`), fetched live — not pretrained, refits on every request. Requires ≥3 months of history or returns `insufficientData: true` rather than fabricating a trend.

**Output:** slope, intercept, R², trend direction, confidence label, 3-month forecast, suggested reorder quantity.

## 2b. Demand Forecast — Trained Neural Network

**Type:** Machine learning — deep learning family. A genuinely trained model with saved weight artifacts, independent of the regression above.

**Architecture:** Feedforward, 3 inputs → 10 hidden units (tanh) → 1 output (linear). Hand-written (no ML library — `brain.js` pulled in a native GPU binding that failed to compile; this has zero dependencies and works identically on Windows dev and Render's Linux host).

**Files:**
| File | Role |
|---|---|
| `backend/src/ml/neuralNetwork.ts` | The network class — forward pass, backpropagation, momentum-based gradient descent |
| `backend/src/ml/generateTrainingData.ts` | Synthetic training data generator |
| `backend/src/ml/trainDemandModel.ts` | Training script (run via `npm run train:ml`) |
| `backend/src/ml/models/demandNet.json` | **The trained model artifact** — saved weights, committed to the repo |
| `backend/src/ml/demandModel.ts` | Inference — loads `demandNet.json`, exposes `predictNextMonthML()` |

**Training data:** 400 synthetic sales sequences (18 months each), mixing base level + linear trend + 12-month seasonality + Gaussian noise — sliding-windowed into **5,095 examples** (3-month input → 1-month target, ratio-scaled to generalize across product volume). Split 85% train / 15% validation.

**Training method:** 2000 epochs, momentum (β=0.9), step-decayed learning rate (0.08 → 0.04 → 0.02), **early stopping** — the checkpoint with lowest validation loss is saved, not the final epoch (validation loss was observed to start rising after ~epoch 200 while training loss kept falling — classic overfitting signal).

**Measured accuracy (real, not claimed):**
| Metric | Value |
|---|---|
| Validation MAPE | ~14–23% (varies slightly per training run — random init + freshly-generated synthetic data each time) |
| Naive baseline MAPE ("predict same as recent average") | ~30–37% |
| Improvement over baseline | ~7–23 percentage points |

**Honest caveat:** Trained/validated on synthetic data generated by us (real seed data has only ~9 months/product — not enough to train a network). When applied to real product history it makes genuine out-of-sample predictions, but real-world accuracy on *your* specific products hasn't been validated the same rigorous way — that needs months of production data.

**Live interactive demo:** Insights page → "What-If Forecast Simulator." Type any 3+ monthly numbers → regression refits client-side instantly, neural network calls `POST /api/insights/ml-predict` (debounced 400ms) and returns its own independent prediction — the two models often disagree, proving neither is faking it.

**Retrain command:** `npm run train:ml` (in `backend/`)

| Feature | AI? | Category | Learns from data? |
|---|---|---|---|
